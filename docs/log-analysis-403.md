# 로그 분석: 403 / 카드가 안 보일 때

## 이 로그에서 확인된 내용

| 구분 | 결과 |
|------|------|
| **GPT API (api.openai.com)** | ✅ **성공** — `response_status=200`, 약 3.4초 후 응답 수신 |
| **백엔드 (127.0.0.1:8080)** | ❌ **403 Forbidden** — `/api/users/me`, `/api/cards` 요청이 403 반환 |
| **스크린샷 처리** | ✅ 동작 — `ScreenshotIndexer_processedAssetIds`에 새 ID 추가됨 (한 장 처리됨) |

즉, **GPT는 정상 응답**하고, **백엔드 API만 403**으로 막히고 있습니다.

## 403이 나는 이유

Spring Security에서 **JWT가 없거나, 만료/서명 불일치**면 인증이 되지 않고,  
`/api/users/me`, `/api/cards`는 **인증 필수**라서 **403 Forbidden**이 반환됩니다.

- 앱이 보내는 `Authorization: Bearer <access_token>` 이  
  - 없거나  
  - 만료됐거나  
  - 백엔드의 JWT 비밀키와 맞지 않으면  
  → 403이 납니다.

## 해결 방법

1. **앱에서 다시 로그인**
   - 로그아웃 후 로그인해서 **새 access_token**을 받고, 그 상태에서 스크린샷 → 카드 가져오기 다시 시도.

2. **백엔드 JWT 비밀키 확인**
   - `application.yml`(또는 환경변수)의 `jwt.secret` 값이  
     **토큰을 발급했을 때와 동일한 값**인지 확인.
   - 서버를 다시 띄우면서 secret을 바꿨다면, 예전에 받은 토큰은 전부 무효가 됩니다.

3. **시뮬레이터/앱 재설치**
   - 예전 토큰이 UserDefaults 등에 남아 있을 수 있으니,  
     앱 삭제 후 재설치 → 다시 로그인 후 동작 확인.

## 참고: 로그에서 보이는 토큰

- `access_token` 이 User Defaults에서 읽혀서 사용되고 있음.
- `exp: 1771334412` (만료 시각) — 서버/앱 시각이 크게 어긋나 있으면 만료로 처리될 수 있음.

요약: **GPT 네트워크/파싱 문제가 아니라, 백엔드 인증(403) 문제**입니다.  
위 순서대로 재로그인과 JWT 설정을 확인하면 됩니다.
