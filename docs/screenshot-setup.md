# 스크린샷 → 카드 연동 설정 (iOS)

## OCR/GPT 결과가 DB에 들어가는 흐름

1. **iOS**: 갤러리 스크린샷 → Vision OCR + Google Vision 레이블 → **GPT 프롬프트로 분류** → `Card` (title, summary, category, subcategory, fields 등)
2. **iOS**: `CardManager.createCard(card)` → **POST /api/cards** (JWT Bearer) 로 전송
3. **백엔드**: `CardController` POST → `CardService.createCard(userNo, body)` → **screenshot 테이블에 INSERT**
4. 이후 **GET /api/cards** 시 DB에서 조회해 카드 목록으로 반환

- 서버 저장에 실패하면(네트워크/401 등) iOS는 로컬만 유지하고, 콘솔에 `❌ CardManager: 서버 저장 실패 → 로컬만 유지` 및 에러 메시지가 출력됩니다.
- 백엔드에서 카드 저장 시 `카드 저장 완료 (userNo=..., title=...)` 로그가 찍힙니다.

## 스크린샷이 카드로 보이게 하는 방법 (요약)

1. **시뮬레이터에서 스크린샷 넣기**  
   시뮬레이터가 포커스된 상태에서 **⌘ + S**를 여러 번 눌러 스크린샷을 찍으면, 사진 앱의 **스크린샷 앨범**에 저장됩니다.
2. **앱 실행** → 로그인 후 **홈** 탭으로 이동.
3. **사진 접근** 권한 허용 (처음 한 번).
4. 카드가 없으면 **「스크린샷에서 카드 가져오기」** 버튼을 누르거나, 화면을 **당겨서 새로고침**.
5. **10~20초 정도 기다리기** (이미지당 GPT 분류 시간). 완료되면 홈에 카드가 나타납니다.

---

홈에 카드가 안 뜨는 경우 아래를 순서대로 확인하세요.

## 1. API 키 설정 (필수)

스크린샷을 GPT로 분류하려면 **OpenAI API 키**와 **Google Vision API 키**가 필요합니다.

- **Xcode**에서 `caplog/Info.plist` 열기
- 다음 키에 실제 값을 넣어주세요 (빈 문자열이면 카드가 생성되지 않습니다).
  - `GPT_API_KEY` → [OpenAI API Keys](https://platform.openai.com/api-keys)에서 발급
  - `GOOGLE_VISION_API_KEY` → [Google Cloud Console](https://console.cloud.google.com/)에서 Vision API 사용 설정 후 발급

키가 없거나 비어 있으면 콘솔에  
`❌ GPT_API_KEY가 Info.plist에 없거나 비어 있습니다`  
같은 로그가 뜨고, 카드는 만들어지지 않습니다.

## 2. 시뮬레이터 갤러리(스크린샷) 연동

- **시뮬레이터**에서는 **Cmd + S**로 스크린샷을 찍으면 **사진 앱 → 스크린샷 앨범**에 저장됩니다.
- 앱 **처음 실행 시** 사진 접근 권한을 허용해야 합니다.
- **홈 탭**에 들어가면:
  - **기존 스크린샷**: 최근 20장을 한 번만 자동으로 가져와서 GPT 분류 후 카드로 만듭니다.
  - **새 스크린샷**: 앱이 켜져 있는 동안 Cmd+S로 찍으면 자동 감지되어 카드가 생성됩니다.

스크린샷이 0개인 상태에서 앱을 켰다가, 나중에 스크린샷을 추가한 경우에는  
**홈 화면을 당겨서 새로고침**하거나, **앱을 다시 실행**하면 최근 20장이 인덱싱됩니다.

## 3. "스크린샷으로 카드가 만들어지고 있는지" 확인하는 방법

- **앱에서**: 카드가 없을 때 보이는 **「마지막 확인」** 박스를 보면 됩니다.
  - 예: `스크린샷 3개 발견 → 처리 시작` → `이미지 로드 완료 (1/3) → OCR/GPT 진행 중...` → `OCR·GPT 완료: "제목" → DB 저장 시도(POST)...` → `POST /api/cards 전송 중...` → **`DB 저장 완료: "제목"`** 이면 서버까지 저장된 것입니다.
  - **POST가 안 나가면**: `파이프라인 중단: OCR/GPT` 또는 `카드는 홈에 추가됨 (서버 동기화 실패)` + 에러 문구가 뜹니다. 서버 실패여도 카드는 로컬에 추가되어 홈·폴더에 보입니다. (권한 없음, 스크린샷 0개, GPT 키 없음, 네트워크/403 등)
- **Xcode 콘솔**: `[Caplog 스크린샷]` 으로 검색하면 같은 단계별 로그가 찍힙니다. POST 실패 시 `❌ POST 실패: ...` 로 원인을 확인할 수 있습니다.

## 4. 확인 순서

1. **Info.plist**에 `GPT_API_KEY`, `GOOGLE_VISION_API_KEY` 값이 들어 있는지 확인.
2. **사진 권한**이 허용되어 있는지 확인 (설정 → Caplog → 사진).
3. 시뮬레이터에서 **Cmd+S**로 스크린샷 1장 이상 찍기.
4. 앱에서 **홈** 탭으로 이동 후, 필요하면 **당겨서 새로고침**.
5. **「마지막 확인」**에서 `DB 저장 완료`까지 나오는지, 또는 콘솔에서 `[Caplog 스크린샷]` 로그를 확인.

이후에도 카드가 안 보이면 **마지막 확인**의 에러 문구나 콘솔의 `❌` 로그를 확인해 보세요.

**카드 생성이 안 될 때 콘솔로 단계 확인:**  
- `[Caplog 스크린샷] OCR·GPT 성공 → 카드 생성 단계` 가 보이면 GPT까지 성공한 것. 그 다음에 `[Caplog 카드] createCard 요청` → `✅ 서버 저장 성공` 또는 `⚠️ 서버 저장 실패 → 로컬 반영됨` 이 나오면 카드는 목록에 추가된 상태입니다 (서버 실패여도 홈에는 뜸).
- `[Caplog 스크린샷] ❌ 카드 생성 불가: OCR 에러` / `GPT 에러` / `GPT 응답 파싱 실패` 가 보이면 그 단계에서 막힌 것입니다. 아래 5·6절 참고.

## 5. "GPT 응답 파싱 실패" 나올 때 (콘솔로 원인 찾기)

Xcode 콘솔에서 아래 로그를 **순서대로** 확인하면, 어느 단계에서 실패했는지 알 수 있습니다.

| 콘솔에서 찾을 문자열 | 의미 |
|----------------------|------|
| `[Caplog GPT] content 타입:` | GPT API 응답의 본문을 문자열/배열 중 어떤 형태로 받았는지. 여기까지 나오면 API 호출은 성공. |
| `[Caplog GPT] ❌ 빈 응답` | API는 200이었지만 `message.content`가 비어 있거나, 문자열·배열 둘 다 아님. (OpenAI 쪽 응답 형식 변경 가능성) |
| `[Caplog GPT 디버그] 1단계:` | 파서가 받은 GPT 원본 텍스트 길이와 앞 200자. **❌로 시작하면** API에서 에러 메시지가 온 것(키 오류, 네트워크 등). |
| `[Caplog GPT 디버그] 실패: GPT가 에러 문자열 반환` | 위와 동일. GPT가 JSON이 아니라 "❌ ..." 문자열을 반환함. |
| `[Caplog GPT 디버그] 2단계:` | 코드펜스(```) 제거 후 JSON 후보 문자열. 여기서 보이는 내용이 `{` 로 시작하는지 확인. |
| `[Caplog GPT 디버그] 실패: JSON 파싱 불가` | 2단계 문자열이 유효한 JSON이 아님. (앞뒤에 설명문, 줄바꿈·따옴표 문제 등) |
| `[Caplog GPT 디버그] 4단계: JSON 키 목록` | 파싱된 JSON의 키들. `category_main`, `title` 이 없으면 다음 단계에서 실패. |
| `[Caplog GPT 디버그] 실패: 필수 필드 누락` | `category_main` 또는 `title` 이 없거나, 문자열이 아님. 4단계 키 목록과 함께 보면 원인 파악 가능. |

**요약:**  
- **1단계에서 이미 ❌** → API 키/네트워크/OpenAI 에러. `📦 HTTP ... body:` 로그에서 API 응답 본문 확인.  
- **JSON 파싱 불가** → GPT가 JSON이 아닌 형태로 답한 것. 2단계 로그에 찍힌 "앞 150자"를 보면 형식 확인 가능.  
- **필수 필드 누락** → GPT가 `category_main`, `title` 을 안 넣었거나 다른 키 이름 사용. 4단계 키 목록과 프롬프트 스키마를 비교.

## 6. "네트워크 오류" / "The network connection was lost" 나올 때

Xcode 콘솔에서 **`[Caplog GPT] 네트워크 에러 상세:`** 로그를 찾으면 **domain, code** 가 찍혀 있습니다.

| code (대표값) | 의미 | 확인할 것 |
|--------------|------|------------|
| **-1005** (NSURLErrorNetworkConnectionLost) | 연결이 중간에 끊김. **시뮬레이터에서 자주 발생**: iOS가 api.openai.com에 QUIC(HTTP/3)로 연결했다가 path가 곧바로 unavailable 되면서 끊기는 경우. | 앱은 **전용 URLSession + 4초 간격으로 최대 4번 재시도**합니다. 콘솔에 `재시도 1/4` 등이 보이면 그대로 기다리기. 그래도 실패하면 Mac 인터넷/VPN/방화벽 확인. |
| **-1001** (NSURLErrorTimedOut) | 요청 시간 초과 | 같은 네트워크에서 Safari로 https://api.openai.com 접속 가능한지 |
| **-1009** (NSURLErrorNotConnectedToInternet) | 인터넷 미연결 | 기기/시뮬레이터에서 웹 브라우저로 인터넷 되는지 |
| **-1200** (NSURLErrorSecureConnectionFailed) | SSL/TLS 실패 | 회사/학교 네트워크에서 HTTPS 차단 여부, 프록시 |

**시도해 볼 것:**  
1. **시뮬레이터** → Mac에서 Safari로 https://platform.openai.com 열어 보기. 안 되면 VPN/방화벽에서 OpenAI 차단 여부 확인.  
2. **실기기** → Wi‑Fi 끄고 LTE로, 또는 다른 Wi‑Fi로 바꿔 보기.  
3. **재시도** → 앱이 자동으로 최대 4번까지 재시도합니다(-1005 시 4초 간격). 콘솔에 `재시도 1/4` 이 보이면 그대로 한 번 더 기다려 보기.

## 실기기(폰)에서 스크린샷이 안 뜰 때

앱은 **갤러리(사진 앱)의 "스크린샷" 폴더**와 연결되어 있습니다 (iOS 스마트 앨범 `.smartAlbumScreenshots`).

1. **설정 → Caplog → 사진**  
   - **「모든 사진」**으로 두면 스크린샷 앨범 전체를 읽을 수 있습니다.  
   - **「선택한 사진」**이면 스크린샷이 포함된 항목/앨범이 선택돼 있어야 합니다.
2. 홈에서 **「스크린샷에서 카드 가져오기」** 버튼을 눌러 다시 가져오기.
3. Xcode 콘솔에서 `📸 ScreenshotIndexer:` 로그 확인  
   - `스크린샷 0개` → 위 1번 권한(모든 사진/선택 항목) 확인.  
   - `사진 권한 없음` → 설정에서 사진 접근 허용.
